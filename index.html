Increment Counter using Vue

How it Works

the three counters below will increment upwards and downwards when the associated button is clicked, all three will increment a different amount. when a counter reaches 0, it will no longer decrement and the reset and decrement buttons will be disabled.

The Code Behind it

The Counter Class 


    export class Counter {
        constructor(selector, initialValue=0) {
            this.count = initialValue;
            this.initialValue = initialValue;
            this.selector = selector;
            this.mount();
        }
    }

the class is initially defined using a "constructor" sets up a given class' properties, and defines the structure of the syntax of the class. The Constructor will be at the top of the class, and will define how the class will connect to the front end.

the "selector" is where we will put that value that connects it to an object in the HTML and stores it. this is most often a HTML element's ID.

"initialValue = 0" is defining that the next value within the string will define the initial value of whatever the class instance contains, and its default is equal to 0.

the line above the constructor is an export function, as this class will be exported. within "js/main.js" the class will be imported like so;


    import { Counter } from './counter.js';

after theclass has been imported, we can create an instance that will define how the class will be used; 

               
    const counter1 = new counter('#objectId', 1);

in this line, "const" creates a new instance, "counter1" gives that instance a name, "new counter" creates a new instance of the counter class, and takes the place of the "constructor", the text in brackets afterwards is equal to the values entered into the constructor, like so; 

  
    constructor(selector, initialValue=0){}
    new counter('#objectId', 1)


the two strings match each other, selector stores the HTML object's ID, in this case the object would have an ID of "#objectID" and the second value in the new counter instance would override the "initialValue" setting it to 1.

the properties within the constructor method define each value for each instance of the counter class. "this.count;" will always have a default equal to the initial value. "this.initialValue = initialValue;" tell the DOM that the initial value of each instance of the counter is defined by that instance within main.js. "this.selector;" will always be defined as the selector. "this.mount();" means that each instance of the class will run the mount method.

The Mount Method 


    mount() {
        const container = document.querySelector(this.selector);
    }

the mount method takes the stored selector value and finds where it is in the HTML document, and adds the counter and buttons to that selected element.

after it finds the selected container, it begins building the increment counter. using the "this" value which defines the specific instance of the class, it appends several properties that create the different elements. the mount method creats a div, several buttons, and appends them to the HTML document through the DOM. after adding them to the DOM, several event listeners wait for those buttons to be clicked before running methods associated with those buttons. the buttons are as follows;

Increment
Decrement
Reset

each button does exactly what their names describe by calling upon associated methods.

The Increment Method 


    increment() {
        this.count++;
        this.decrementButton.disabled = false;
        this.update();
    }

The way the first counter increments, decrements and resets are defined within the methods that follow the initial mounting of the counter class.

Incrementing is done by using an incrememnt method that contains "this.count++;" within this line of code, "this" refers to the instance of the class being used, "count" is a special property that stores the current vlaue of the counter. the two plus signs following the count method changes its function, making it increase the count by 1 every time the increment method runs.

the second line of the increment method activates the decrement button. this is done because we do not want the counter to go below zero, and further methods will deactivate the decrement button when the counter hits zero.

the last line within this method "this.update();" calls upon an additional method that updates the current state of the counter, and will be described below.

The Decrement Method 


    decrement() {
        if (this.count==0){
            this.decrementButton.disabled = true;

            return;
        }
        this.count--;
        this.update();
    }

the decrement method is similar to the increment method, but has more functionality. it begins with an "if" statement that state that if the counter equals 0, disable the decrement button, like so;

if (this.count==0){
this.decrementButton.disabled = true;
return;}

the return value exits the method entiry upon determining that the count is at 0. if the count is not equal to 0, this section of the decrement section is ignored, and the rest of the method runs. "this.count--;" decreases the count by 1, and "this.update();" then runs the update method.

The Reset Method 


    reset() {
        this.count = this.initialValue;
        this.update();
    }

the reset method is very simple, its first line "this.initialValue = initialValue" seems confusing but what this line is saying is that the initial value of each instance of the class is defined by the initial vlaue of that instance, and remember we are defining the initial value of each instance in main.js. the final line of the reset method; "this.update();" calls upon the update method to run.

The Update Method 


    update() {
        this.display.textContent = `Count: ${this.count}`;
        this.decrementButton.classList.toggle('disabled', this.count===0);
        this.resetButton.classList.toggle('disabled', this.count===0);
    }

the update method's purpose is to update the count following any of the other methods, its what changes the number on the screen. every method redfines the "count" of each instance of the Counter class using "this.count" and the first line of this method changes the display's text content to a different number using a template literal. the two following lines toggle the "disabled" class on the reset button and the decrement button when the count hits zero, all that class does is turn the background of each button grey, which signals to the user that they are not currently active.

The StepCounter Subclass 


    export class StepCounter extends Counter {
        constructor(selector, initialValue = 0, step) {
            super(selector, initialValue);
            this.step = step;
        }
    }

A subclass is a variant of a class, the reason why we are making a subclass instead of a new class is that all methods of a subclass are the same as it's parent class, unless redefined. so it essentially copies the Counter Class' functions and allows us to make tweaks and changes to them in order to suit our needs. this is done on the first line of this subclass, where "export class Stepcounter extends Counter" this line exports the subclass so it may be imported later and the "extends" syntax marks it as a child of the Counter class. from here we can start refining the class.

the constructor on this class adds another value to the defining string, a "step" value which will be defined later. "super" is a property that tells the new constructor to inherit the values of its parent's constructor, note that earlier it was explained that the methods are carried over, but the values are not unless a super is used. in the brackets following the super we define what values we are choosing to carry over.

the final line of this begins to define "step" and says that the value of every instance of "step" within the Subclass is equal to the definition of step that is defined in Main.js, displayed below, the first value after the id selector is the "initialValue", and the second value is the "step" value. we will work to define what the "step" is by redefining methods. 


    import { Counter, StepCounter } from './counter.js';

    const counter1 = new Counter('#counter-container1');
    const counter2 = new StepCounter('#counter-container2', 5, 5);
    const counter3 = new StepCounter('#counter-container3', 10, 10);

The Redefined Methods 


    increment() {
        this.count +=this.step;
        this.decrementButton.disabled = false;
        this.update();
    }

    decrement() {
        if (this.count===0){
            this.decrementButton.disabled = true;

            return;
        }
        this.count -= this.step;
        this.update();
    }

within the subclass we will redefine increment and decrement whilst leaving the reset and update methods the same. we leave reset and update the same because they need not change, reset changes the count to the initial value which we are defining in main.js, and update handles updating the count with the new value defined by increment and decrement, so it also does no need to change as the increment and decrement methods define how much the counter is changing by.

you will also notice that the increment and decrement methods are very similar to what they were initially, instead of making them increase by 1 by using "++" or "--" we are changing it by an amount defined in main.js, which was decided in the StepCounter's constructor. all these functions do it say, change the amount to this subclass' instance by an amount given by the constructor, nothing else is changed.
